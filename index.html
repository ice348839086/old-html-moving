旧页面迁移 / TrojanUI 概念验证

不重写旧业务逻辑，也能迁移老内网页面。

这个演示用三个页面展示同一个想法：原始旧系统、基于影子 DOM 的现代界面接管，以及直接插入 HTML 导致失败的对照组。

预览 1

原始旧系统界面

展示旧 ERP 风格的表格布局、全局宋体、棕色文字，以及恶意全局样式污染。
预览 2

TrojanUI 实验组

把深色现代卡片界面挂载到影子 DOM 中，同时把用户意图路由回旧系统函数。
预览 3

直接插入对照组

不使用影子 DOM，直接把现代界面塞进旧页面，因此会被旧系统样式明显污染。

推荐测试流程

  1. 打开 原始旧系统界面，先看基线效果。
  2. 打开 TrojanUI 实验组，点击 查询库存，确认旧系统弹窗能拿到现代输入框的值。
  3. 查看底部状态面板，确认 legacy:response 已经把旧系统结果回写到现代界面。
  4. 打开 直接插入对照组，对比没有影子 DOM 时的样式污染失败效果。
